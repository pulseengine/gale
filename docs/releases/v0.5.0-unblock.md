# gust v0.5.0 — "The Unblock & Consolidate"

**Status:** planning. This is the release-scope north star; each workstream lands as
its own PR and closes its rivet scope before the tag is cut.

## Why this is a big release

Two things converged. **(1) The version line needs reconciling:** the last actual
release tag is **gale v0.4.0** (2026-07-16), but a large body of work has merged to
`main` since and was only *labelled* with versions in PR titles / rivet `release:`
fields — it was never tagged. **(2) Every tracked upstream blocker just closed at
once** (2026-07): `meld#326`/`#334`, `synth#686`, `kiln#415`, `gale#149`. So v0.5.0
both *ships the accumulated merged work* and *takes the leap the blockers were
gating*.

## Scope

### A. Consolidate the already-merged, unreleased work (into the v0.5.0 tag)
Everything merged after v0.4.0 gets released here — verify each is on `main` and its
rivet scope is closed before including it in the notes:
- I-ISO MPU region-swap; partition-switch demonstrator + Health-Monitor / cross-sensor
  voting; timer-sleep (tickless component sleep); the **verified driver suite**
  (adc/dac/pwm/i2c/can/dma closed) + certificate-checked BV obligations; the **wdg**
  driver + its **silicon** anchor; the **AADL target model** (PR #206) with generated
  constants/`memory.x`/WIT, **silicon-validated on real STM32G474 + STM32F100**.

### B. Syscall seam — the headline capability (unblocked by meld#326)
The stateful `gust:os` providers (**log / channel / io / spawn**) dissolved. This was
gated for months on the meld shared-memory corruption bug (`meld#326`), now fixed.
- Adopt a meld build carrying the `#326` fix; confirm how gale invokes meld + whether
  it needs pinning.
- Recover the prior WIP (app-tl / app-ts crates + `gust_os_tl_probe` / `gust_os_ts_probe`
  + `os-node/*.o`) — check `feat/gust-v0.4.0-syscall-seam`, `git stash`, branches.
- Re-run the syscall probes → **expect uncorrupted output** (was corrupted before the
  fix — the kill-criterion). Then close the V: Kani over the seam FSM, `rivet
  VER-OS-SYSCALL-001`, Renode content-gate.

### C. Toolchain modernization (enabling + a real perf win)
> **Byte-check measured 2026-07-22:** re-dissolving wdg-thin with synth 0.49.0 /
> loom 1.2.0 produced **text 648 B vs the committed 660 B** (~2% smaller, 0-SRAM
> preserved) — no adoption blocker, codegen improved (consistent with synth#686).
> The `.o` differs across the 38 versions, so the committed cores + byte-identical
> Renode gates must be regenerated and every content-gate re-verified (register
> effects must stay identical) before bumping the pins.

- **synth** 0.11.50 → 0.49.0 — ships `synth#686` (measured **12 % on gust_mix**).
- **loom** 1.1.14 → 1.2.0; **meld** to the `#326`-fixed build.
- Re-dissolve gust cores; **byte-check** the `.o`s and re-verify every Renode
  content-gate (the *register effects* must stay identical even if the `.o` bytes
  change — that is the safety-relevant invariant). PR-gated, measured; update the
  `.github/workflows` pins (`release.yml`, `release-wasm.yml`) only once verified.

### D. Silicon breadth via the target model
Validate more of the driver suite on **real F100** through the new target model —
prefer self-contained checks (no external wiring), e.g. **ADC reading the internal
Vrefint** (channel 17). Add each peripheral's base to `stm32f100.aadl`, build the
per-driver F100 firmware on generated constants, flash via the Pi (openocd
`stlink-hla.cfg`), capture. Extend from wdg to the driver suite.

### E. Docs accuracy + reader-friendliness (release hygiene)
A release ships honest, readable docs. Fold in the claim-verification pass:
- **Reconcile version claims** — remove/repair any "v0.5–v0.10 released/shipped"
  language that predates an actual tag; state plainly what is tagged vs merged.
- Fix stale claims (perf numbers from the old toolchain; "verified" where it is
  dissolved-not-proven; counts) per the verification-honesty rule.
- Make the top-level README/overview newcomer-friendly (a plain "what is this" TL;DR;
  gloss the jargon: dissolve, meld/loom/synth, thin-seam, gust:hal) — **without**
  inflating capability.

## Sequence (dependencies)
`E-versions` (reconcile the version story) → **C** (toolchain; changes bytes anyway
and B needs the newer meld/synth) → **B** (syscall seam) → **D** (silicon breadth) →
**E-docs** final pass → **A** consolidation + tag **v0.5.0**.

## Exit criteria (the tag is cut only when all hold)
1. All workstream PRs merged green (never over a real red; rerun infra flakes with a
   root-cause, don't dismiss).
2. rivet: every `approved`/`implemented` artifact in scope has its V closed
   (`rivet validate` clean, 0 new diagnostics); syscall seam `verified`.
3. Toolchain adoption is byte-checked and every Renode content-gate re-verified.
4. Silicon: the drivers claimed silicon-validated actually ran on a real board
   (evidence in `RESULTS-*.md`); everything else honestly scoped "qemu/Renode, not
   silicon".
5. Docs: no stale version/perf/verification claim survives; README passes a
   newcomer read.
6. Dissolves ≠ verified: every dissolved artifact is probe- or Kani-gated.

## Kill-criterion for the release
If the syscall probes still show corruption after adopting the `#326`-fixed meld, B is
NOT done — the blocker isn't actually cleared; stop and re-file upstream rather than
shipping a corrupted seam.
