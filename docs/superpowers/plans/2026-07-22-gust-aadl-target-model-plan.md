# gust AADL Target Model — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make gust's board/SoC/device facts a spar/AADL model that a gale-side build-time generator turns into per-target constants, linker `memory.x`, and a gust:hal WIT world — proving it end-to-end (full V) on the IWDG driver across two live boards (G474RE / G4 and F100 / F1).

**Architecture:** AADL is the single source of truth (spar parses + EMV2-analyzes it). A gale-side generator (`tools/gust-target-gen`) reads `spar items --format json` and emits committed, DO-NOT-EDIT artifacts under `benches/gust/targets/generated/`, guarded by a CI drift gate. `gust_wdg_silicon` is retargeted to consume the generated constants — the keystone parity + silicon oracle. rivet closes the trace.

**Tech Stack:** spar (AADL v2.3, installed at `~/.cargo/bin/spar`), Rust (generator + firmware), WIT (gust:hal seam), rivet 0.23.0, Kani (existing driver proofs, unchanged), probe-rs (G474RE) + openocd/ST-LINK-V1-HLA on the Pi (F100).

## Global Constraints

- **Zero runtime cost:** the target model is *build-time only*. Nothing here adds a runtime instruction, a TCB atom, or SRAM. Generated constants bake into the object.
- **Base-16 AADL literals only:** write register/memory addresses as `16#4000_3000#`, never `0x4000_3000` (spar#337 — `0x` silently parses to `0`).
- **Generated files are committed** with a first-line `// GENERATED by gust-target-gen from <model>.aadl — DO NOT EDIT` header, and are never hand-edited. A CI drift gate re-generates and asserts `git diff --exit-code`.
- **No third-party company/product names in public artifacts** beyond board/part identifiers already established in this repo (STM32F100/G474, Cortex-M, ST-LINK are used in existing merged files; do not add company names or new third-party product names to PR bodies, issues, or committed docs). spar issues stay genericized.
- **Do not modify the driver FSMs.** `wdg-thin/src/lib.rs` and its Kani proofs are unchanged; only the *constant bindings* consumed by the silicon firmware move to generated code.
- **spar dialect facts:** a `system` may contain `processor`/`memory`/`device` subcomponents (legal AADL); spar enforces category rules at `instance`/`analyze`. EMV2 goes in an `annex EMV2 {** … **};` block (parsed as opaque text today — see `safety/aadl/semaphore.aadl:153`).
